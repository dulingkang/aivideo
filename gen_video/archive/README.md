# 文档归档说明

本目录用于归档已解决或过时的修复和优化文档。

## 归档原则

1. **已完全解决的问题**：相关修复已合并到代码中，文档仅作历史参考
2. **重复的文档**：内容已合并到其他文档中
3. **过时的优化方案**：已被更好的方案替代

## 目录结构

- `fixes/` - 已解决的修复文档
- `optimizations/` - 已实施的优化文档
